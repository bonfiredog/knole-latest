ex_audio_copy

Creates a copy of an existing sound

Syntax

ex_audio_copy(soundName, copyName)
Argument Description
soundName The original sound name
copyName The new sound name, must be unique
Returns the ID of the new copied sound in exAudio sound database

Description

There are some times that you need to have a different instance of the same audio resource but you need it to have the exact same settings of an existing sound and classes. This can save you having to manually change a sound's properties and re-add it to any classes. The new sound name must be unique otherwise the function will fail.

Example

//Copy the sound "fx_jump" and create new sound "fx_jump2" with the same settings
ex_audio_copy("fx_jump", "fx_jump2");

Minimum Supported Version

exAudio 0.8.0
